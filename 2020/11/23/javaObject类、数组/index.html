三. 面向对象

  • Object类

  • 查找Object类中的方法：源码/Java类库帮助文档

    int hashCode() //获取对象哈希值

    boolean equals(Object dbj) //判断两个对象是否相等

    String toString() //将对象转换成字符串形式

    protected void finalize() //垃圾回收期负责调用的方法

  • API（Application Program Interface）：

    应用程序编程接口，整个JDK类库就是一个javase的API，每个API都会配置一套API帮助文档

1. toString()方法

SUN建议所有子类重写toString()方法

2. equals方法

Java中比较基本数据类型数据是否相等用“==”，比较引用数据类型统一用equals方法

  • equals方法的目的：判断两个对象是否相等

  • String类的toString()方法和equals方法已经重写

    比较字符串时不能用“==”，必须用equals方法

  • equals方法多层重写

3. finalize()方法

  1. 当一个对象即将被垃圾回收器回收时，垃圾回收器负责调用此方法，不需要手动调用

  2. 垃圾销毁时机：希望在对象销毁时机执行的代码写在finalize()方法中

    对比静态代码块：在类加载时执行，且仅执行一次

  • 垃圾回收器（GC）不是轻易启动的，垃圾太少或时间没到都可能不启动

4. hashCode()方法

  • hashCode()方法返回的是哈希码：是一个对象的内存地址经过哈希算法得到的值。执行结果可以看做一个对象的内存地址

5. 内部类

  • 内部类：在类的内部定义的类（可读性差，尽量不用）

    静态内部类：类似与静态变量

    实例内部类：类似于实例变量

    局部内部类：类似与局部变量

  • 匿名内部类是局部内部类的一种

    不建议使用匿名内部类：代码无法复用、可读性差

四. 数组

  1. Java中数组是一种引用数据类型，数组对象存储在堆内存，父类是Object
  2. 数组是数据的集合，是一个容器
  3. 数组中可以存储基本数据类型数据，也可以存储引用数据类型的数据
  4. 数组中存储“Java对象”时，实际上存储的是对象的引用（内存地址）
  5. Java中规定，数组创建后长度不可变
  6. 数组分类：一维、二维、三维、…多维（一维数组使用较多，二维较少）
  7. 所有数组对象都有length属性（Java自带），用来获取数组中元素个数
  8. 数组中的元素都有下标，下标从0开始，最后一个元素的下标是length - 1，对数组中元素进行存取时都需要通过下标进行
  9. Java要求数组中元素类型统一：int类型数组只能存储int类型，Person类型数组只能存储Person类型
  10. 存储时数组中的元素内存地址是连续的
  11. 数组中首元素的内存地址作为整个数组对象的内存地址

  • 数组数据结构的优点：

    查询/检索某个下标的元素的效率高（可以说是查询效率最高的一种数据结构）

    因为：①每个元素的内存地址是连续的 ②每个元素类型相同，所占空间大小相同 ③知道第一个元素地址和每个元素内存空间大小就可以计算出某个下标元素的内存地址

  • 数组缺点：

    1. 随机删除或增加元素时效率低：为保证每个元素内存地址连续，删除/增加元素会导致其他元素统一向前/后位移（对数组最后一个元素的增删没有效率影响）
    2. 数组不能存储大数据量：很难在内存空间上找到一块特别大的连续内存空间

  • 一维数组

    声明/定义一维数组语法格式：

  • 初始化一维数组：

    已知数组中每个元素时用静态初始化；元素未知时用动态初始化，后期赋值

  • 方法参数类型为数组

  • main方法的String[] args数组：

    1. JVM调用main方法时，会自动传一个String数组过来

    2. 数组对象的长度默认为0，不是null

    3. String[] args用来接收用户输入参数：用户在控制台输入参数，会被自动转换为String数组

      例如：运行java MainTest abc def xyz，JVM自动将“abc def xyz”按空格分离，转换为数组{“abc”,”def”,”xyz”}存储在String[] args数组中

  • 模拟系统登录
  • 数组中存储引用数据类型

  • 一维数组扩容、拷贝：

    新建一个大容量数组，然后将原来数组中的元素依次拷贝到大数组中

    数组扩容涉及到拷贝问题，效率较低，在开发中尽可能少用数组拷贝

  • 二维数组

    1. 二维数组是一个特殊的一维数组，这个“一维数组”的每个元素都是一个一维数组（三维数组是一个特殊的二维数组，这个“二维数组”的每个元素都是一个一维数组）
