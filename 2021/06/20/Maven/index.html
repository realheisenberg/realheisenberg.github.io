Maven

  • 软件开发的阶段：

    需求分析：分析项目功能，如何实现

    设计阶段：设计项目使用什么技术

    开发阶段：编写代码，编译、自我测试

    测试阶段：由测试人员进行测试

    打包、发布阶段：给用户安装项目

  • Maven的功能

  1. 项目的自动构建：帮助开发人员做项目代码的编译，测试， 打包，安装，部署等工作

  2. 管理依赖（管理项目中使用的各种jar包）

    依赖：项目中使用的其他资源，常见类型是jar包。项目使用mysql驱动，即项目依赖mysql驱动

  • Maven：项目的自动化构建工具，管理项目依赖（不使用Maven需要手动下载指定版本jar包、手动处理jar文件之间的依赖）

  • Maven的安装：

    下载地址：http://maven.apache.org/，本文使用apache-maven-3.3.9-bin.zip

  1. 确认JAVA_HOME环境变量指向jdk安装目录
  2. 解压文件，解压后的文件路径不要有中文和空格
  3. 把maven安装目录中下的bin的路径添加到path环境变量中：E:\Maven\apache-maven-3.3.9\bin
  4. 测试，在命令行输入mvn -v

  • maven解压后目录结构：

    bin：maven可执行命令

    conf：maven配置文件

Maven核心概念

Maven项目约定的目录结构：

Maven可以独立使用或与idea一起使用

POM文件

POM：Project Object Model项目对象模型，Maven把项目当做模型处理

Maven通过pom.xml文件实现项目构建和依赖的管理

1. 坐标

坐标由groupId、artifactId、version组成

坐标是资源的唯一标识，在maven中每个资源都是坐标，坐标简称gav

groupId：组织名称、代码，是公司或团体的标识。通常为域名倒写，也可以是域名倒写+项目名称

aitifactId：项目名称，如果groupId包含项目名称，此处为子项目名

version：项目版本号（三位组成：主版本号.次版本号.小版本号）如果版本号带有-SNAPSHOT，表示为快照，不是稳定版本

packing：项目的打包类型，有jar、war、ear、pom等，默认为jar

项目中使用gav：

  1. 每个maven项目都有自己的gav
  2. 管理依赖，需要jar包的gav作为标识

搜索坐标的地址：https://mvnrepository.com/

2. 依赖

依赖使用dependencies和dependency标签管理，以及gav坐标

maven使用gav作为标识，从互联网下载依赖的jar包到本机上

3. 仓库

maven仓库存储各种gar包：maven工具自己的jar包、第三方jar包例如mysql驱动、自己写的jar包

仓库的分类：

  1. 本地仓库：本机磁盘的某个目录，默认路径是登录操作系统账号的目录中/.m2/reposisitory

    修改本地仓库位置：

    1.创建新目录，例如E:\openrepository

    2.修改maven安装路径\conf\setting.xml文件，添加一行<localRepository>E:/openrepository</localRepository>

    3.把仓库资源拷贝到E:\openrepository目录下

  2. 远程仓库：需要通过互联网访问

    1.中央仓库：一个ftp服务器，所有开发人员共享的仓库

    2.中央仓库镜像：中央仓库的备份，各大城市都有镜像

    3.私服：公司内部maven服务器，通过局域网访问

maven自动使用仓库，首先访问本地仓库，如果没有，访问私服或镜像

4. Maven生命周期

Maven生命周期包括：清理、测试、报告、打包、安装、部署

插件：完成项目的各个阶段，需要通过插件执行maven命令，插件就是jar包，一些类

命令：执行功能需要命令，例如mvn compile

  • 单元测试（junit）：

    junit是一个单元测试工具，用于开发人员自测。java中单元指的是方法，方法是测试的最小单位

  1. 加入junit依赖：

  1. 在src/test/java目录下创建测试文件

    测试类类名一般是Test+要测试类的类名

    测试类包名要与被测试类包名一样

    测试方法：public、没有返回值，方法名建议使用test+测试的方法名、方法没有参数

    测试类和测试方法都可以单独执行

    测试方法上加注释@Test

5. 命令

  1. mvn clean：清理命令，删除之前生成的数据（删除target目录），使用maven-clean-plugin插件

  2. mvn compile：编译命令，编译src/main/java目录下的java源代码，使用maven-compiler-plugin插件

    同时把class文件拷贝到target/classes目录。classes是存放类文件的根目录（也叫做类路径：classpath），使用maven-resources-plugin插件

  3. mvn test-compile：编译命令，编译src/test/java目录下的源代码，使用maven-compiler-plugin插件

    把生成的class文件和src/test/resourses目录中的文件拷贝到target/test-classes目录，使用maven-resources-plugin插件

  4. mvn test：测试命令，执行test-classes目录下的测试程序，使用maven-surefire-plugin插件

  5. mvn package：打包，把src/main目录下所有class和配置文件打包为压缩文件，放在target目录下。

    默认为jar文件，web应用为war文件。打包与test无关，使用maven-jar-plugin插件

    打包生成的文件名格式：artifactId-version.packaging

  1. mvn install：把生成的打包文件安装到maven仓库，使用maven-install-plugin插件
6. 自定义配置插件

可以在pom.xml文件中使用build标签设置插件

idea中使用Maven

设置idea使用自己安装的Maven：

  1. 打开File-Settings-Build,Excution,Deployment-Build Tools-Maven：

    设置Maven home directory为Maven安装目录：E:/Maven/apache-maven-3.3.9

    设置User setting file为Maven的配置文件目录，勾选右侧Override：E:\Maven\apache-maven-3.3.9\conf\settings.xml

    设置Local repository本地仓库：E:\openrepository

  2. 打开File-Settings-Build,Excution,Deployment-Build Tools-Maven-Runner：

    设置VM Options不用下载archetype-catalog.xml文件：-DarchetypeCatalog=internal

    设置JRE：选择本机1.8jdk

对新建或导入的项目配置Maven：

​ 打开Edit-Other Settings-Setting for New Progects，其余操作与上面相同

    创建空工程，New Module，选择org.apache.maven.archetypes:maven-archetype-quickstart，勾选Create from archetype选项

    修改pom.xml文件：

  • 基于Maven创建web项目：

    选择org.apache.maven.archetypes:maven-archetype-webapp

依赖管理、常用设置

在pom.xml中使用scope标签设置依赖的范围，表示该依赖在项目构建的哪个阶段起作用

scope值：

​ compile：默认，参与项目构建的所有阶段

​ test：在测试阶段起作用，例如junit

​ provided：部署到服务器时不需要提供jar包，由服务器提供。例如servlet、jsp的依赖

  • properties中的设置

  • 全局变量

    在properties中自定义标签，标签内容就是变量的值

    使用全局变量语法：${变量名}

  • 资源插件

    maven默认把src/main/resources中的文件拷贝到target/classes目录

    maven默认只处理src/main/java中的.java文件，编译为字节码文件并拷贝到target/classes目录，不处理其他类型文件
