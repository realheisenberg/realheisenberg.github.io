16. MVC概念

MVC：Model模型、View视图、Controller控制器

MVC最早出现在JavaEE的Web层，指导Web层代码有效分离

View视图：只负责数据和界面的展示，不接收与显示数据无关的代码（JSP/HTML）

Controller控制器：只负责接收请求，调用业务层代码处理请求、派发页面（Servlet）

Model模型：将与业务逻辑相关的数据封装为具体的JavaBean类（JavaBean/domain/entity/pojo）

MVC思想目的：降低耦合度

17. Cookie和Session

1. Cookie

Cookie是服务器通知客户端保存键值对的一种技术，客户端每次都将Cookie发送给服务器，每个Cookie大小不超过4kb

  • Cookie生命控制：setMaxAge()方法控制Cookie被销毁的时间

    传递参数为正数（int）时，表示在指定的秒数后被销毁

    为负数时，表示在浏览器关闭时销毁

    为0时，表示立即删除Cookie

  • 设置Path有效路径

    Cookie的Path属性可以过滤出哪些Cookie发送给服务器，哪些不发

    例如：

    CookieA path=/工程路径

    CookieB path=/工程路径/abc

    当请求地址为http://ip:port/工程路径/a.html时，CookieA发送，CookieB不发送

    当请求地址为http://ip:port/工程路径/abc/a.html时，CookieA和CookieB都发送

2. Session

Session是一个接口（HttpSession）

Session意为会话，用来维护客户端和服务器之间关联的一种技术

每个客户端都有一个自己的Session会话，经常用来保存用户登录后的信息，Session对象保存在服务器内存中

Session技术底层是基于Cookie技术的

  • 创建和获取Session
  • Session域数据的存取
  • Session生命周期控制

  1. public void setMaxInactiveInterval(int interval)方法：设置Session的超时时间（单位为秒）

    传入参数为正数时，设置Session超时时长；为负数时表示永不超时（极少使用）

  2. public int getMaxInactiveInterval()方法：获取Session的超时时间

  3. public void invalidate()方法：让当前Session会话立即失效

Session超时：客户端两次请求的最大间隔时长（在没有超出时长时再次请求，将重新计时）

在浏览器没有Cookie的情况下向服务器发请求（即使之前的Session没有超时，但已被删除），服务器会创建一个新的Session会话，返回Sessionid给客户端，浏览器创建新的Cookie对象

18. Filter过滤器

Filter过滤器是JavaWeb三大组件之一（Servlet、Listener、Filter）

Filter过滤器是JavaEE规范（接口）

Filter作用：拦截请求、过滤响应

拦截请求的应用场景：权限检查、日记操作、事务管理等

Filter的生命周期

  1. 构造器方法：Web工程启动时执行（Filter创建）
  2. init初始化方法：Web工程启动时执行
  3. doFilter过滤方法：每次拦截到请求时执行

FilterChain过滤器链

FilterChain.doFilter()方法的作用：

  1. 执行下一个Filter过滤器（如果有下一个Filter）
  2. 访问目标资源（没有下一个Filter）

多个Filter过滤器执行的特点：

  1. 所有Filter和目标资源默认在同一个线程中
  2. 多个Filter共同执行时，使用同一个Request对象
  3. Filter的执行顺序由web.xml中从上到下配置的顺序决定

ThreadLocal

ThreadLocal可以解决多线程的数据安全问题

ThreadLocal可以给当前线程关联一个数据（可以是任何变量、对象）

  1. ThreadLocal可以为当前线程关联一个数据（可以像Map一样存取数据，key是当前线程）
  2. 每个ThreadLocal对象只能为当前线程关联一个数据（只会保存最后一次赋值），要关联多个数据就需要多个ThreadLocal对象
  3. ThreadLocal对象定义时，一般都是static类型
  4. ThreadLocal中保存的数据会在线程销毁后，由JVM自动释放

21. i18n国际化（了解）

国际化是指：同一个网站可以支持多种不同的语言（使用同一个网址）

i18n：Internationalization，以i开头，以n结尾中间18个字母

国际化三要素

  1. Locale对象

    Locale表示不同的时区、位置、语言

    常用的有：zh_CN中国，中文、en_US英文，美国

  2. Properties属性配置文件

    命名规则：baseName_locale.properties

    中文配置文件名：i18n_zh_CN.properties、英文配置文件名：i18n_en_US.properties

  3. ResourceBundle资源包

    ResourceBundle.getBundle()方法：根据给定的baseName和Locale读取相应配置文件，得到文字信息

    ResourceBundle.getString(key)方法：得到不同国家语言信息
